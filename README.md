# Spooner Labs

Personal site for publishing experiments and thoughts in systems, technology, organizations, and whatever else is worth exploring.

**Live at:** [spoonerlabs.com](https://spoonerlabs.com)

## Stack

Plain HTML output generated by a lightweight Python build script. No framework. Deploys automatically to Cloudflare Pages from `main`.

## Structure

```
/
├── build.py               # Static site builder
├── requirements.txt
├── site.css               # Shared stylesheet (source)
├── logo.svg               # Logo (source)
├── templates/             # Jinja2 page templates
│   ├── base.html
│   ├── index.html
│   ├── about.html
│   ├── contact.html
│   ├── speaking.html
│   ├── experiments.html
│   └── projects.html
├── data/                  # Content and config
│   ├── site.yaml          # Global config (email, social links)
│   ├── experiments.yaml
│   ├── projects.yaml
│   └── speaking.yaml
└── dist/                  # Built output (Cloudflare Pages serves this)
```

## Local development

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python build.py
```

Then open `dist/index.html` in a browser.

## Adding content

**New experiment:** add an entry to `data/experiments.yaml`, run `python build.py`.

**New project:** add an entry to `data/projects.yaml`, run `python build.py`.

Fields for experiments and projects:
```yaml
- title: "Title"
  description: "Short description."
  category: "Category"
  status: live | in-progress | idea | archived
  link: "https://..."   # optional
```

## Publishing

Build locally, then push — Cloudflare Pages serves `dist/` directly with no build step:

```bash
python build.py
git add dist/
git commit -m "Build"
git push
```

Cloudflare Pages settings:
- **Build command:** *(leave empty)*
- **Output directory:** `dist`
